2026 — Q2

2026-05-21 Keep the Node API surface; rehost the Sheets export (do not cut)
Chose
Keep api-server.js / google-sheets-client.js / POST /api/export-to-sheets as a local/dev-only tool; document it as not-in-prod; open a follow-up to rehost the export so it works against the static Pages surface.
Why
The Sheets export is a real, needed function for faculty workloads (and a desirable backup/institutional archive path for schedules). Cutting it would delete a wanted capability; it is just mis-hosted for a static-Pages world.
Rejected
Cut entirely (deletes a needed function); keep-as-is without a rehost plan (leaves the export permanently broken in prod).
2026-05-21 Docs variant mirrors the Foundry shell, not the dashboard theme
Chose
System sans-serif, SF Mono, square edges, hairline borders, blue interactive accent — matches css/public-schedule.css.
Why
Reorientation docs are read alongside the public surface; a single visual vocabulary keeps the product feel coherent.
Rejected
Sora display font, dark-gradient header, EWU-red accent (program-command dashboard theme) — wrong family for long-form reading.
2026-05-19 Path B over Path A on RLS hardening
Chose
Editor allowlist (public.editors + is_editor()) on top of existing "Public read" policies; closed PR #246 with a pointer.
Why
Path A's tenant migration assumed tables that didn't exist in prod (pathways, pathway_courses) and would have zero-result'd the editor app without seeding user_programs first. Path B locked writes the same week without that risk.
Rejected
Path A tenant migration (PR #246). Deferred, not abandoned — retry requires SQL hardening, an end-to-end exercise against the develop project, and a user_programs seed plan.
2026-05-19 Editor allowlist mutable only via direct SQL
Chose
Four rows in public.editors; no UI, no API, no self-service add/remove.
Why
The allowlist is the only thing standing between an authenticated user and write access to prod data; a UI surface would be a new attack/mistake vector for a control with four total entries.
Rejected
Admin UI for editor management — explicitly out of scope until editor count or churn justifies the complexity.
2026-05-19 Email signups disabled at Supabase project level
Chose
Toggle off in the Supabase dashboard; verified by probe returning 422 signup_disabled.
Why
The editor allowlist is the only write gate. Open signup would let any address create an auth.users row — harmless on its own, but a future bug that grants editor-by-default would instantly become exploitable.
Rejected
Leave signups open and rely on the allowlist alone — defense in depth is cheap here.
2026-05-19 Manual workflow_dispatch deploy, gated on per-SHA CI
Chose
Remove push: branches: [main] trigger; deploy job's first step queries the deployed commit's test + onboarding check-runs and aborts unless both are success.
Why
Auto-deploy-on-push is the load-bearing risk for a solo maintainer supervising agents; a per-SHA gate also prevents a later un-CI'd commit from riding a stale green.
Rejected
"Most recent CI run" gate — defeated by a second commit landing between PR merge and dispatch.

2026 — Q2 planning (2026-05-15)

2026-05-15 Phase 2 hard-locked behind Phase 1 docs
Chose
Units 6–9 (docs variant, ORIENT, decision log, ONBOARDING) complete before develop rescue or strategic-PR triage starts.
Why
Intermittent solo maintainer; cold-reorientation must be guaranteed before the largest cross-branch merge work begins. Adversarial review flagged a stall risk; risk accepted.
Rejected
"Minimum-viable docs" fallback that would let Phase 2 start early — explicitly declined to keep the lock real.
2026-05-15 Reorientation docs as HTML in a docs design-system variant
Chose
Author ORIENT / decision-log / ONBOARDING as HTML under docs/, styled by a new css/docs.css + the existing css/design-system.css.
Why
Standing maintainer preference; consistent with the no-build constraint; lets the prod-state stamp be rewritten mechanically by the deploy job instead of a comment convention.
Rejected
Markdown in docs/ — would not render in the product's visual vocabulary and would require a separate viewer.
2026-05-15 Data safety lives at RLS + deploy, not at the branch boundary
Chose
Treat the prod Supabase anon key as public-by-design; rely on RLS + the manual deploy gate as the data-safety controls.
Why
The anon key is committed in js/supabase-config.js — a separate develop branch was never a real isolation boundary, just a perceived one. Reality-aligned model.
Rejected
Keep gitflow under the belief that develop protects production data.
2026-05-15 Trunk-based single main; retire develop
Chose
Move agents and humans to short-lived branches off main; delete develop after its forward work is rescued.
Why
Gitflow drifted develop ~2 months from main; PRs rotted; the maintainer couldn't tell solved from stale. Trunk removes the drift category entirely.
Rejected
Repair gitflow with discipline — declined as a non-structural fix to a structural problem.
2026-05-15 Retire Replit references; GitHub Pages is the prod runtime
Chose
Documented prod path matches reality — GitHub Pages, static, no Node server in production.
Why
Replit autoscale and api-server.js are not in the production deploy; leaving them in CLAUDE.md / README misleads cold readers.
Rejected
Keeping the Replit docs as historical context — superseded by this log entry.
